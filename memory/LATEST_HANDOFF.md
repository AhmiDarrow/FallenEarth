## Current Focus: v0.6.0 follow-up polish COMPLETE

The user's brief — "cooking table like other crafting tables will be craftable, recipe to craft one unlocks at level 5. proceed with polish" — landed as: (1) a new `cooking_table` item (category: `station`, max_stack 1), (2) a craftable recipe at L5 (4 withered_branch + 2 iron_ore + 1 teal_crystal → 1 cooking_table), (3) LocalMapGenerator auto-places a cooking table in every start hex at spawn + 8, +8 (marked occupied before resource emitters), (4) `assets/sprites/stations/cooking_table.png` generated by a new `tools/generate_station_sprites.py` PIL script, and (5) item icons for the 5 new items generated by an extended `tools/generate_item_icons.py` (added `station` category).

**Key polish insight:** Use `FileAccess.file_exists` (not `ResourceLoader.exists`) when checking for newly-generated PNGs. Godot's resource cache is stale until next editor reimport, so freshly-written sprite files appear missing in headless tests even though they're on disk. Fixed in `smoke_cooking.gd` for the new tests.
