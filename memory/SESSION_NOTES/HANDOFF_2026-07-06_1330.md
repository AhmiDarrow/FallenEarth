# v0.10.11 Combat Visual Polish (round 2)

## User request
"GETTING CLOSER" — the v0.10.10 changes (square grid + always-visible
borders + size-aware sprite scaling) made the cells distinguishable
and sprites fit, but the 7×7 grid was still too big: the top row was
hidden behind the TopPrompt banner and the bottom row was nearly
touching the End Turn / Retreat action bar. Move-range highlights were
also invisible — the white 0.22-alpha tint over light sand terrain
read as no highlight at all.

## What shipped

### CELL_SIZE 56 → 40 (grid fits the screen)
- `BattleGridView.CELL_SIZE = 40` (was 56)
- `BattleCell.CELL_SIZE = 40`
- `BattleUnit.CELL_SIZE = 40`
- `BattleBackground.TILE_SIZE = 40` (for the grid-rect math)
- `BattleBackground.buffer = 50` (was 60; sized for the smaller 280px grid)
- 7×7 grid = 280px (was 392px). Now:
  - Grid top: y=360-140=220 (was y=164)
  - Grid bottom: y=360+140=500 (was y=556)
  - TopPrompt bottom: y=172 → **48px clear above**
  - ActionBar top: y=560 → **60px clear below**
  - No more overlap with TopPrompt or ActionBar

### Visible cyan move highlights
- `BattleCell.COLOR_MOVE` = `Color(0.30, 0.85, 1.0, 0.40)` (was white
  1.0/0.22 — invisible against light sand/ground)
- Reads as a clear "this tile is in move range" tint without
  washing out the underlying terrain
- ATTACK alpha also bumped: 0.45 → 0.50 (more visible red)
- SKILL alpha also bumped: 0.45 → 0.50 (more visible purple)

### Sprite target 46 → 32 (fits the smaller 40px cell)
- `BattleUnit._load_sprite` target_px = 32 (was 46)
- 128×128 human at 0.25x = 32px (was 0.359x = 46px)
- 64×64 mob at 0.5x = 32px (was 0.719x = 46px)
- 20×20 placeholder at 1.2x = 24px (was 1.5x = 30px)
- All three fit the 40px cell with ~4px margin per side

### Smoke test updates
- `_test_v102_cell_sizing`: expects all three CELL_SIZE = 40
- `_test_v1010_cell_square_positioning`: cell (2,3) at (80, 120),
  base centered at (20, 20)
- `_test_v1010_unit_square_positioning`: cell center (100, 140)
- `_test_v1010_move_highlight_opacity`: expects cyan (b >= r) with
  alpha in [0.30, 0.55]
- `_test_v1010_unit_sprite_scales_to_cell`: target_px = 32, math
  confirms 128×128 → 32px and 64×64 → 32px
- `_test_v1010_decor_smaller_and_buffered`: buffer = 50 (was 60)
- `smoke_combat_v100.gd`: setup() / setup_from_data() use 40
