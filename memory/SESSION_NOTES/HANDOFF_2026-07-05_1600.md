## 3. Key Decisions & Rationale

- **Recipe ingredients (4 wood + 2 metal + 1 crystal):** feels mid-game appropriate. Wood is plentiful from the start; iron_ore requires mining (L1 with Stone Pickaxe); teal_crystal requires the Neon Bogs geode nodes. The recipe gates on the player's progression naturally.
- **Recipe station: `none`** (not `cooking_table`): you craft a cooking_table BEFORE you have a cooking_table. Chicken-and-egg. So it's in the basic inventory tab from L5. The existing cooking recipes (cooked_meat etc.) still require `station: "cooking_table"` and only show in the station UI.
- **Item `stackable: false`, `max_stack: 1`:** the cooking table is a placeable structure, not a stackable consumable. One per inventory slot. If the player wants a second one, they craft a second one.
- **Auto-placement at spawn + 8, +8 (not 5,5):** the offset is large enough that the table is visually distinct from the spawn pocket (24-tile cleared area) but small enough that the player finds it within a few seconds of walking.
- **`_emit_start_cooking_table` is per-hex, not just first-hex:** the player can craft their own cooking tables at L5 to place in other hexes, but each generated local map gets a free one for the auto-spawn. This matches the v0.4.0 spec ("place one of each near the start hex in every world").
- **Sprites generated by procedural PIL scripts, not PixelLab:** matches the v0.3.0 convention (`tools/generate_tiles.py`, `generate_resource_nodes.py`, etc.). No API cost.
- **Filesystem check (`FileAccess.file_exists`) instead of `ResourceLoader.exists`:** newly-generated PNGs may not be in Godot's resource cache until the next editor reimport. `FileAccess.file_exists` checks the filesystem directly. Updated both the cooking_table sprite test and the item-icons test.

## 7. Context That Must Survive

- **`_emit_start_cooking_table` always emits one table per hex.** The player can craft their own via the L5 recipe. Each generated local map has its own cooking table at spawn + 8, +8. Reloads of the same hex don't move the table.
- **The cooking table cell is marked occupied BEFORE the resource/floor-pickup emitters run.** So no resource node or floor pickup can spawn on the table's cell.
- **Item icons are generated by PIL, not PixelLab.** `tools/generate_item_icons.py` and `tools/generate_station_sprites.py` are the source of truth. Re-run them when items.json / station ids change.
- **`FileAccess.file_exists` over `ResourceLoader.exists` for new PNGs.** Resource cache is stale until next reimport. This is the same pattern the existing `HarvestNode._refresh_sprite` uses (with a fallback path).
- **The `station` item category is new.** Other placeable structures (future Worktable, ArmorTable, Blacksmith kits) should also use `category: "station"`. The icon generator handles them with the cauldron-shaped icon; if you want a different silhouette, extend `STATION_STYLES` in `tools/generate_station_sprites.py`.
