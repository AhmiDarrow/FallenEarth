## 3. Key Decisions & Rationale
1. **AnimatedSprite2D always, not dual Sprite2D/AnimatedSprite2D path** — cleaner code, single code path. Static PNGs become single-frame SpriteFrames dynamically
2. **v3 mode for animations** — 1 gen/direction, good quality, fast. Template mode was used only as fallback for scrap_crawler's death (v3 kept failing)
3. **keep_first_frame=true** — animation frame 0 is the character's rotation reference, giving 5 frames for 4-animated (idle) or 7 for 6-animated (walk/attack/death). Matches the existing mob `.tres` format
4. **MCP tools for PixelLab** (not direct REST) after discovering the MCP tools work reliably with small (64×64) reference images — the 180×180 originals caused base64 truncation in MCP transport
5. **Template animation fallback** for scrap_crawler's death (v3 kept failing at 95%) — used `falling-back-death` template instead
