## Key Design Decisions

1. **Dictionary-based RACE_SPRITES** — Changed from flat `race → path` to nested `race → {male: path, female: path}`. This allows gender-based selection while keeping the mapping organized.

2. **Default to male** — When gender field is missing or empty, falls back to male sprite. This maintains backward compatibility with existing NPC data.

3. **No API calls needed** — Existing base sprites (128x128) already contain all 8 directions in a 2×4 grid. The AtlasTexture extraction works for both male and female sprites without needing to generate rotations via PixelLab API.

4. **Gender variety in room data** — Distributed genders across races for visual diversity: 5 male, 5 female NPCs.
