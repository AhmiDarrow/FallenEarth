# v0.10.10 Square Grid Fix

## User request
"COMBAT STILL HAS GRAPHICAL ISSUES AND DOES NOT LOOK RIGHT" — screenshot showed:
1. The whole 7x7 grid was rotated 45° into a giant diamond shape.
2. Each cell was also a diamond polygon, with a dark brown floor
   that covered up the actual terrain sprite.
3. The terrain tile (24x24) was barely visible inside each diamond.
4. Move-range highlight was a 0.42-alpha white wash that hid the
   terrain underneath completely.
5. Decor props (boulders, skulls, cacti) were 22px blocks at the
   grid corners that read as "competing cells", not scenery.
6. The Blight Toad sprite was half off the grid (iso offset bug).
7. The legacy right-side MainVBox (action buttons + status +
   turn order + log) was bleeding through into the middle of
   the screen — the player saw the old action bar floating
   between the grid and the bottom UI.

## What shipped

### Square grid (revert iso-diamond)
- `BattleGridView.cell_to_world(x, y) -> (x*56 + 28, y*56 + 28)`
- `BattleGridView.cell_to_iso(x, y)` is now an alias for
  `cell_to_world` (back-compat for code that still called it).
- `_iso_grid_bounds()` is now a square `Rect2(0, 0, 392, 392)`;
  the grid_layer is offset by `-Vector2(196, 196)` to center
  the grid in the BattleGridView's local space.
- 7x7 grid at 56px = 392x392px (~30% of 1280x720 viewport).

### Square cells (revert diamond cells)
- `BattleCell` no longer draws a `Polygon2D` diamond floor.
- The cell IS the terrain sprite: a 56x56 `Sprite2D` with a
  `clip 24x24 region` of the biome's terrain atlas, scaled
  `56/24 = 2.33x` with `TEXTURE_FILTER_NEAREST` (no rotation).
- Highlight is a `ColorRect(56, 56)` on top of the terrain
  (was a `Polygon2D` diamond in v0.10.5+).
- `BORDER_THICKNESS` = 2 (was 3; on 56px cells the chunky 3px
  line read as busy). Still readable as a deliberate frame.
- `_height_label` Label removed — height is now encoded as a
  modulate tint on the base sprite (subtle, no extra label
  clutter on the grid).
- `_build_blocked_x` (was `_build_blocked_diamond_x`) — two
  crossed Line2Ds at the cell center with a 6px padding.
- `_build_square_outline` — 4 Line2Ds around the cell edges for
  the border-only ATTACK/SKILL highlights.

### Move highlight opacity reduced
- `BattleCell.COLOR_MOVE.alpha` = 0.22 (was 0.42).
- The terrain tile is now clearly visible underneath the
  move-range tint, instead of being washed out.

### Decor props shrunk + buffered
- `BattleBackground.DECOR_COUNT` = 14 (was 22).
- `grid_rect buffer` = 60px (was 12px) — decor no longer
  lands on top of cells.
- Corner clusters reduced from 3-5 per corner to 2-3 per
  corner, with anchor distance bumped from 40-100 to 80-160.
- Decor `base_scale` uses `randf_range(1.05, 1.45)` (was
  `0.85-1.10`); with the 24/64 base scale, that's ~26-35px
  rendered — well below the 56px cell size, so decor reads
  as small scenery, not competing blocks.
- Decor rotation reduced from `±0.4 rad` to `±0.15 rad` —
  decor no longer looks like "tilted playing cards".

### Legacy right-side MainVBox hidden
- `TacticalCombat._ready()` now sets
  `legacy_main.visible = false` and
  `legacy_actions.visible = false` so the right-side
  status / turn order / log / action buttons (which the
  TopPrompt + TurnOrderBar + SkillBar + bottom action bar
  all replaced) stop bleeding into the middle of the screen.
- The `@onready` refs to `status_label`, `turn_order_label`,
  `instructions_label`, `log_label` are kept (invisible) so
  other code paths that touch them still resolve.

### BattleUnit square positioning
- `setup_from_data()` places the unit at the cell center
  (`x*56 + 28, y*56 + 28`), not the iso-transformed position.
- `move_to()` tweens to the same square target.
- `set_grid_pos()`, `play_attack_swing()` all use the square
  cell-center math.

### CombatFeedback (HP bars) square positioning
- `setup_hp_bars()` places each HP bar at the cell center,
  offset upward by 24px so it sits just above the unit sprite.
- HP bars no longer land half-off the grid (was a 0.25 iso
  vertical step that pushed them above the diamond row).

## Architecture constraints held
- `CELL_SIZE` is a single `const` on each combat node (still
  56 in `BattleGridView`, `BattleCell`, `BattleUnit`).
- `cell_to_world` is a single function; bumping CELL_SIZE
  cascades through positioning.
- The `cell_to_iso` alias is kept (returns the same as
  `cell_to_world` now) so any code that still calls it
  (the targeting reticle, combat popups) doesn't break.
