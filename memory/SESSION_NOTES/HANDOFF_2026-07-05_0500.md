## 2. What Was Accomplished

### Settlement pattern
- **SettlementNode** (`scripts/SettlementNode.gd` + `scenes/SettlementNode.tscn`) — a Node2D on the world map representing a settlement structure. Sized per template (small 4×4, medium 6×6, large 10×10 tiles). Has a `setup(town_data)` method. The LocalMapView places one of these per town from `world_data.towns_seeded`.
- **Settlement** (`scripts/Settlement.gd` + `scenes/Settlement.tscn`) — the interior instance loaded when the player enters. Single-room placeholder with: title (town name), faction label, **NPC list** (party-joinable NPCs the player can Invite), services list (tavern / trader / worktable / quest_board / faction_hq etc., each with a "Talk" button), and a "Leave settlement" button. Esc / E also exits.
- **SettlementManager** (`scripts/SettlementManager.gd`, autoload) — `enter_settlement(hex_key, hub)` / `leave_settlement()` / `is_inside_settlement()`. Owns the active settlement interior lifecycle. Signals `entered_settlement` and `left_settlement`.

### One-settlement-per-hex rule
- **WorldGenerator._place_towns** in `scripts/WorldGenerator.gd`:
  1. Places Riftspire on a random good-start candidate.
  2. Places NPC towns: 1 per ~25 hexes (clamped 4-10), even percentage of `data/factions.json`, no two axial-adjacent (distance >= 2), distance >= 2 from Riftspire, on good-start candidate tiles only.
  3. One `large_hub` per faction; the rest are `medium_settlements`, with a few `small_outposts` for flavor.
  4. Each placed town is tagged on the underlying tile (`is_town`, `town_faction`, `town_template`).
- The rule is enforced upstream (in WorldGenerator). The view layer (`LocalMapView.get_settlement_at`) returns the first found, with a comment noting that production code ensures uniqueness upstream.

### Party NPCs spawn in settlements
- `Settlement._resolve_resident_npcs()` picks a subset of `PartyNPCManager.available_npcs` deterministically by town index: 1 NPC for small_outpost, 2 for medium_settlement, 3 for large_hub. Same town → same residents.
- The Settlement interior shows each resident as a row with an **Invite** button. Clicking Invite calls `PartyNPCManager.invite(npc_id)` — the NPC moves from `available_npcs` to `party_members` and the row disappears from the interior.

### Shop + Mission givers
- **ShopInterface** (`scripts/ui/ShopInterface.gd`) — buy/sell interface. Default stock: bandages, sticks, stones, iron ore, withered branches (each with a buy price). EC is debited via `ProgressionManager.spend_ec`. Selling 1 of an item at half the buy price (or 1 EC minimum).
- **MissionBoardInterface** (`scripts/ui/MissionBoardInterface.gd`) — lists offers from `MissionManager.get_offers_for_npc` for each party member. Clicking Accept calls `MissionManager.accept_mission`.
- Both are opened from inside the Settlement interior — "Talk" on the trader building opens ShopInterface; "Talk" on the quest_board building opens MissionBoardInterface.
- All other buildings (tavern, worktable, etc.) show a stub greeting for now; full interaction lands in a follow-up.

### Wiring
- `scripts/HubWorld.gd` — `_adjacent_settlement_hex()` looks at the SettlementNode on the player's adjacent cells. `_try_enter_settlement()` checks Riftspire L50 gate (via `TownManager.can_enter_riftspire(player_level)`) and either opens the settlement or shows a message. The E key disambiguates: if a settlement is adjacent, enter it; else if a HarvestNode is adjacent, gather; else open the Equipment tab. Esc inside the settlement leaves.
- `scripts/LocalMapView.gd` — added `settlement_layer` (Node2D, y-sorted). `_populate_settlements(towns)` instantiates one `SettlementNode` per town. New `get_settlement_at(cell)` query.
- `scenes/LocalMapView.tscn` — added `SettlementLayer` node between PickupLayer and MobLayer.

### Modified

- `scripts/GameState.gd` — no changes (world_data already supports arbitrary keys). `world_data.towns_seeded` and `world_data.riftspire_hex_key` are written by WorldGenerator.
- `scripts/Settlement.gd` — new NPC list UI; "Invite" buttons; per-building "Talk" actions that route to ShopInterface / MissionBoardInterface / greeting stub.
- `validate_scripts.gd` — added 6 new scripts to the script list (TownManager, SettlementManager, SettlementNode, Settlement, ShopInterface, MissionBoardInterface).

## 3. Key Decisions & Rationale

- **Settlements mirror the player base pattern** (per the user's spec). World map has a SettlementNode (large structure). Press E adjacent → load Settlement interior. Press E / Esc inside → leave. The same `_unhandled_key_input` E-key disambiguation handles both.
- **One settlement per world hex** (per user). Enforced in `WorldGenerator._place_towns` (each town is placed on a unique hex from `placed_hexes`; no two axial-adjacent). The view layer just renders one SettlementNode per town.
- **NPC residents in settlements** (per user). The settlement interior shows a deterministic subset of `PartyNPCManager.available_npcs` (1-3 depending on template size). The Invite button moves them from `available_npcs` to `party_members`. In Phase 5 the real procedural spawn lives inside the settlement, and the test NPCs are replaced.
- **Shops + Mission givers live in settlements** (per user). Two new UIs (ShopInterface, MissionBoardInterface) are openable from the settlement's "Talk" buttons. The trader building → ShopInterface; the quest_board building → MissionBoardInterface. Other buildings show stub greetings for now.
- **Riftspire is gated at L50.** `TownManager.can_enter_riftspire(player_level)` returns false below 50. HubWorld's `_try_enter_settlement` checks this and shows a message via `_show_settlement_message` (Phase 3 stub; Phase 8 adds a proper toast).
- **Interior placement is a placeholder.** Real interiors (multiple buildings, traveling NPCs, mini-quests) land in a follow-up. For Phase 3 the Settlement is a single room with text panels + buttons.
- **The 3 station UIs (WorktableUI / ArmorTableUI / BlacksmithUI) are still TODO.** The plan said these would be separate UIs triggered by E-press near a station. Currently the Worktable / Armor Table / Blacksmith buildings in a settlement just show a stub greeting. The ShopInterface serves as a single-shop template; the station UIs will mirror that pattern in a follow-up.
- **NPCs in the settlement are a UI proxy for the real spawn system.** The Settlement reads `PartyNPCManager.available_npcs` and renders a subset. The real spawn is Phase 5 (procedural NPCs per world). For Phase 3 the PartyNPCManager seeds 3 hard-coded test NPCs which all 3 settlements share (each town shows a deterministic subset).

## 7. Context That Must Survive

- **Tile cell size is 24×24.** Maps are 512×512 cells.
- **One settlement per world hex.** Enforced in `WorldGenerator._place_towns`. Each town has a unique hex from `placed_hexes`. View layer (`LocalMapView.get_settlement_at`) returns the first found at a cell.
- **Settlement is the same pattern as the player base** (Phase 6). Exterior structure on the world map; press E adjacent to enter the interior; press E / Esc inside to leave.
- **Party NPCs spawn in settlements.** The Settlement interior shows a deterministic subset of `PartyNPCManager.available_npcs`. Invite button → `PartyNPCManager.invite(npc_id)`. Real spawn lands in Phase 5.
- **Shops + Mission givers in settlements.** Trader building → ShopInterface (buy/sell). quest_board building → MissionBoardInterface (browse/accept). Other buildings → stub greeting.
- **Riftspire is gated at L50.** `TownManager.can_enter_riftspire(player_level)`. HubWorld's `_try_enter_settlement` checks this and shows a message via `_show_settlement_message` (Phase 3 stub).
- **SaveManager has NO `class_name`.** Use `var sm: Node = get_node_or_null("/root/SaveManager")` and call methods on the instance.
- **No `Array.any(func(...))`.** GDScript 4.3's `Array.any()` expects a Callable; lambdas as bare functions trigger a "Trying to assign value of type 'Callable' to a variable of type 'Array'" error. Use a for loop.
- **JSON load via `data` property:** `if "data" in raw: data = raw.data`.
- **Backups folder is excluded** from Godot (`backups/.gdignore`) and from git (`.gitignore`).
