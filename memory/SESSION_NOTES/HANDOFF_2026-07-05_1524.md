## Key Design Decisions

1. **AtlasTexture for sprite extraction** — Godot's native `AtlasTexture` cleanly extracts a 16x16 frame from the 128x128 spritesheet without image processing. No custom shader or crop needed.
2. **Fallback to colored dots** — If a race sprite is missing or invalid, NPC renders as colored dot (same as Phase B). Backward compatible.
3. **Biome floor via Sprite2D** — Each floor cell gets a `Sprite2D` with the biome's ground tile. At 24px cells with 24x24 tiles, they tile naturally. `TEXTURE_FILTER_NEAREST` keeps pixel art crisp.
4. **Faction accents on top/bottom walls only** — Subtle visual differentiation without overwhelming the room. Side walls stay dark for contrast.
5. **Race field in room data** — NPC race is specified per-room (not per-settlement) so the same room template can show different NPC appearances in different settlements.

## Context That Must Survive

- `RoomView.setup()` signature is now `setup(room_data, start_x, start_y, biome, faction)` — the extra params are optional with defaults
- `RACE_SPRITES` maps visual_tag names (from races.json) to sprite paths — must match `data/races.json` `visual_tag` fields
- Character sprites are 128x128 with 8 directions in 2 rows of 4. Each frame is 16x16. South = row 0, col 0.
- `FACTION_WALL_ACCENTS` keys use snake_case faction names (e.g. `"iron_accord"`) — must match faction data from WorldGenerator
- Biome floor textures are 24x24 ground tiles from `assets/tilesets/{biome}/ground.png` — same tiles used by the overworld TileMapLayer
