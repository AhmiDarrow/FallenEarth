## 2. What Was Accomplished

### New script
- **`scripts/HoverTooltip.gd`** — extends `Control`. Tracks `_current_target: String` + `_hover_start_time: int`. 1-second dwell (`DWELL_MS = 1000`). Follows the mouse at `MOUSE_OFFSET = Vector2(14, 14)`. Outline-styled text (white + 3px black outline + size 14) for legibility on any background. Public API: `update(mouse_pos, target_text)`, `get_current_target()`, `is_showing()`.

### Modified
- **`scripts/HubWorld.gd`**
  - Added `HoverTooltipScript` preload + `_hover_tooltip: Control` member.
  - `_setup_hover_tooltip()` — instantiates the tooltip and adds it to the HubWorld tree with `MOUSE_FILTER_IGNORE` so it doesn't steal mouse events.
  - `_tick_hover_tooltip()` — runs each frame in `_process`; converts mouse to local coords, calls hit-test, updates tooltip.
  - `_hit_test_at_world(world_pos)` — does the actual hit-testing with this priority order:
    1. Resource node (highest priority; shows the node's `name` from `node_data`)
    2. Floor pickup (shows item's display name from InventoryManager)
    3. Mob marker (looks up display name in `data/mobs.json`)
    4. Rift marker ("Rift")
    5. NPC marker (NPC name from `_get_npc_at_hex`)
    6. Mission marker ("Mission")
    7. Terrain label (always falls through here)
  - New helpers: `_terrain_label_at_cell`, `_mob_name_at_cell`, `_resolve_mob_display_name` (reads `data/mobs.json` for the display name), `_npc_name_at_hex`.
  - Player's own cell is excluded (no point showing "Player" all the time).

## 3. Key Decisions & Rationale

- **Tooltip is a Control, not a CanvasLayer:** simpler. The HubWorld itself is a Control, and the tooltip renders on top of the world_grid as a child. The tooltip's `mouse_filter = IGNORE` so it doesn't steal mouse events.
- **HubWorld owns the hit-test logic:** the tooltip is purely a "render this text at this position" widget. HubWorld has the world state (resource_nodes, mobs, NPCs, rifts, terrain) and does the lookup. Keeps world knowledge out of the UI script.
- **Hit-test priority matches gameplay expectations:** resource nodes and floor pickups (interactive gameplay) are highest priority; mobs (combat) and rifts (dungeon entry) are next; NPCs and missions (dialog/quests) follow; terrain label (informational) is the fallback.
- **Player's own cell excluded:** avoids the tooltip constantly showing "Player" or its underlying terrain.
- **Lookup of mob display name via `data/mobs.json`:** the underlying `sprite_id` is technical (e.g. `blight_toad`); players should see the lore-friendly name (e.g. "Blight Toad"). Helper `_resolve_mob_display_name` reads the JSON.
- **No .tres.gd` for the tooltip:** the tooltip is a script-only Control (no .tscn file). It's instantiated via `HoverTooltipScript.new()` in HubWorld. Simpler than a scene file for a one-purpose UI widget.
- **Mob level included in label:** "Blight Toad (Lv.3)" gives players at-a-glance info about whether a mob is dangerous.

## 7. Context That Must Survive

- **Tile cell size is 24×24.** Maps are 512×512 cells.
- **Atlas is 4 rows** (ground=0, debris=1, vegetation=2, blocked=3).
- **@onready doesn't fire in SceneTree._initialize** — smoke tests use `await process_frame` after `add_child`.
- **JSON load via `data` property:** in Godot 4.3, `load("res://file.json")` returns a JSON instance, not a Dictionary. Access via `.data`. Pattern: `if "data" in raw: data = raw.data`.
- **Backups folder is excluded** from Godot (`backups/.gdignore`) and from git (`.gitignore`).
- **GDScript strict-type narrowing:** after `if bucket is Dictionary:`, the next `elif bucket is Array:` triggers a compile error because the type system has narrowed `bucket` to `Dictionary`. Use a fresh `var bucket = data.get(...)` to widen the type if you need both checks.
- **HoverTooltip updates with `update(mouse_pos, target_text)`** — empty string means "no target, hide".
- **Dwell is 1 second.** Target switch immediately hides; same target after 1s shows.
- **Mouse offset is (14, 14)** — below-right of cursor, so it doesn't sit under the cursor.
- **Hit-test priority order** is documented in §3 of this handoff.
- **HubWorld owns hit-test, tooltip owns rendering.** The tooltip script knows nothing about the world.
- **InventoryManager is the source of truth** for item display names. `_hit_test_at_world` calls `InventoryManager.get_item_name(item_id)` for floor pickups.
