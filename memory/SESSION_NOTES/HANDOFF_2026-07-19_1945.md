## 2. What Was Accomplished

### Phase A — Spawn Architecture Fix
- **Root cause identified:** `HubWorld._seed_local_mobs()` used `EncounterBuilder.generate_procedural_enemy()` for ALL mobs. That method always produces humanoid NPCs (race: Human/Chthon/Nullborn/Vesperid) with human appearance data — non-humanoid mobs from `mobs.json` were ignored.
- **Fix:** Rewrote `_seed_local_mobs()` into 3 lanes:
  - **Lane 1** (MobSpawner): beasts & mounts from `mobs.json` using `sprite_id`→`assets/mobs/{id}.png`
  - **Lane 2** (EncounterBuilder): humanoid procedural enemies (raiders, cultists, etc.)
  - **Lane 3** (reserved): rift creatures
- **Density:** 6–18 mobs per hex (6-10+6×danger beasts, 0-2+2×danger humanoids)
- Added `wildlife_class` property to all 47 original mob entries in `mobs.json` (beast/mount/rift_beast)
- Added `wildlife_class` to all 10 archetypes in `enemy_archetypes.json` (beast/humanoid)
- Added `allowed_classes` filter to `MobSpawner.gd` (`pick_mob_template()` and `_build_pool()`)

## 3. Key Decisions & Rationale
1. **Added `wildlife_class` as property flag on existing JSON entries** rather than a separate category file — minimal schema change, keeps mobs.json as source of truth
2. **Three explicit spawn lanes** (not one unified spawner) — preserves EncounterBuilder for humanoids with appearance/procedural generation, MobSpawner for sprite-based beasts, clear separation of concerns
3. **Placeholder sprites first, PixelLab later** — created colored-shape placeholders immediately so game can run, then replaced with PixelLab art as jobs completed
4. **SKITTISH as a State (not a substate)** — vermin flee takes priority over aggro; once safe they resume WANDER. Simplified state machine
5. **1-direction PixelLab objects** for wildlife — mobs are small overworld sprites, don't need 8-direction rotation. Keeps generation cost low (1 gen per creature instead of 8)
