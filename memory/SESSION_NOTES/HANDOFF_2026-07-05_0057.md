## 3. Key Decisions & Rationale

- **Why drop rift_scar entirely:** The user said the orange scars "look like old icons" and asked to lose them. Rifts are still a core game feature (procedural RiftInstance, 75% boss, close returns to upworld) ‚Äî they just render as markers now, not as a tile type.
- **Why keep normalization in `LocalMapView` instead of in `LocalMapGenerator`:** Old saves might still have `terrain[i] == 4`. Handling it at the view layer means: (a) we don't need to migrate save data, (b) any other code path that produces a map_data works without modification, (c) the smoke test pins the behavior.
- **Why use a 4-row atlas not a 5-row with an empty row:** Empty rows waste GPU memory and break the natural `set_cell(coord, 0, terrain)` API (which expects `atlas_coords.y` to be a valid row index). The compact 4-row atlas is correct Godot 4.3 practice.
- **Why the `.gdignore` in backups/:** Godot scans the entire `res://` tree when building the class registry. Backups contain old scripts with `class_name` declarations; without `.gdignore`, Godot picks up duplicates and emits "Class X hides a global script class" errors that break compile. The `backups/` directory is intentionally outside the runtime project; `.gdignore` is the right Godot mechanism to mark it as not-for-runtime.

## 7. Context That Must Survive

- **Tile cell size is 24√ó24** (`TileSetService.CELL_SIZE`); the local map is 512√ó512 cells = 12,288√ó12,288 px. Camera centers on `_local_x * 24 + 12`, `_local_y * 24 + 12`.
- **Atlas is 4 rows** (ground=0, debris=1, vegetation=2, blocked=3). BLOCKED has full-cell collision on physics layer 0.
- **`MobVisual` parent is `MobLayer`** (y-sorted).
- **`@onready` doesn't fire in `SceneTree._initialize` for a freshly added node** ‚Äî the smoke test uses `await process_frame` after `add_child` to ensure `_ready` has run before calling `configure()`.
- **Backups folder is excluded from Godot** via `backups/.gdignore`. Without this, Godot scans the older scripts in backups and emits "Class X hides a global script class" errors that break compile. **The .gdignore is load-bearing**; if you re-create backups/ make sure the .gdignore is preserved.
- **Backups folder is excluded from git** via `.gitignore` (line added in Phase 0). Backups go to `backups/YYYY-MM-DD_HHMM_<label>/` and stay local-only.
- **LF/CRLF warnings during `git add`** are normal on Windows. Godot saves files in CRLF on Windows; git normalizes to LF on commit. No action needed.
- **TileSetService.TERRAIN_BLOCKED = 3** (not 4 as it might appear if a refactor mis-counts). The smoke test asserts the BLOCKED cell renders at atlas coord `(0, 3)`.
- **Legacy map_data normalization:** any `terrain[i] < 0` or `terrain[i] > TERRAIN_BLOCKED` is treated as ground. This handles the removed rift_scar=4 value without needing a save migration.
