# v0.10.10 Combat Visual Polish

## User request
"COMBAT STILL HAS GRAPHICAL ISSUES AND DOES NOT LOOK RIGHT" — screenshot
showed the 7×7 grid reading as a single mass of cells with no
distinguishable borders, and the player character (Recruit) sprite
overflowing its cell dramatically. Enemy units (Blight Toad) had the
same overflow problem at the 64px native sprite level.

## What shipped

### Always-visible thin cell borders
- `BattleCell._highlight_border.visible = true` (was `false` —
  only shown on ATTACK/SKILL)
- Default border color: `Color(0.18, 0.16, 0.12, 0.55)` (dim
  warm gray)
- ATTACK highlight: border brightens to `COLOR_ATTACK` (red)
- SKILL highlight: border brightens to `COLOR_SKILL` (purple)
- MOVE / CURSOR / NONE: border stays at the dim default
- `set_highlight` no longer toggles border visibility — only the
  border color. The 4 Line2D edges stay drawn at all times.

### Size-aware unit sprite scaling
- `BattleUnit._load_sprite` no longer hard-codes `scale = 0.7x`
- New: `var target_px: float = 46.0` (~82% of 56px cell with margin)
- `var native_max: float = maxf(tex_size.x, tex_size.y)` from the
  loaded texture's get_size()
- `var scl: float = target_px / native_max`
- Result:
  - **128x128 human portrait** (was 0.7x = 89.6px, overflowed
    the 56px cell) → now 0.359x = **46px** ✓
  - **64x64 mob sprite** (was 0.7x = 44.8px, fine) → now 0.719x
    = **46px** ✓ — same visual size as before, just computed
  - **20x20 placeholder** (was 1.5x = 30px) → unchanged

### Smoke test updates
- `_test_battle_cell_border_highlight`: updated to expect border
  visible at HIGHLIGHT_NONE (was the old behavior)
- `_test_v102_cell_sizing`: BORDER_THICKNESS expectation updated
  from 3 (chunky) to 1 (thin)
- New `_test_v1010_border_always_visible`: 3 checks — border
  visible at default, MOVE keeps border, ATTACK keeps border in red
- New `_test_v1010_unit_sprite_scales_to_cell`: 4 checks — source
  contains target_px=46, contains scl formula, hard-coded 0.7x
  removed, math renders 46px for both 128x128 and 64x64

## Architecture constraints held
- CELL_SIZE is still a single `const CELL_SIZE = 56` on each of
  BattleGridView, BattleCell, BattleUnit. Bumping it cascades
  cleanly to the cell border thickness (1px) and the unit sprite
  target size (46px) without magic numbers in the layout code.
- Unit sprite scaling is now size-aware: any future sprite
  (96x96 boss portrait, 192x192 hi-res character) will auto-fit
  the 46px target without code changes.

## Files modified
- `scripts/combat/BattleCell.gd` — border always visible, default
  border color, set_highlight no longer toggles border visibility
- `scripts/combat/BattleUnit.gd` — size-aware sprite scaling
- `tools/smoke_combat_polish.gd` — 2 new test groups + updates to
  existing HIGHLIGHT_NONE and BORDER_THICKNESS expectations
