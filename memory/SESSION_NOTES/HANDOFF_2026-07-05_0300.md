## 2. What Was Accomplished

### New autoloads
- **`scripts/ProgressionManager.gd`** — XP, level (max 256), EarthCoin (EC) tracking. `add_xp(amount)` triggers level-ups (curve: `50 + level * 25` XP per level). `add_ec` / `spend_ec` for currency. `get_snapshot` / `restore_from_snapshot` for save/load.
- **`scripts/LootRoller.gd`** — `roll(mob, biome)` returns `{item_drops, xp, ec}`. `roll_and_apply(mob, biome, inv, prog)` also writes the result to InventoryManager and ProgressionManager. XP = `mob.level * 5 + 5`; EC = `mob.level * 2 + rand(1, 5)`. Per-mob drops (if `mob.drops` is present) take priority over the biome loot table.

### New UI scripts
- **`scripts/ui/HUD.gd`** — top-level in-game HUD overlay. Composes: top bar (player name/race/class/level/EC), HP/MP/XP bars, Minimap, Hotbar, "≡ Menu" button. Connects to InventoryManager / ProgressionManager signals. `notify_cell_changed()` triggers a Minimap refresh.
- **`scripts/ui/Hotbar.gd`** — 10-slot Control with one Button per slot. Keys 1-0 (KEY_1..KEY_9, KEY_0) call `select_slot(0..9)`. Selected slot is highlighted; `equipped_item_id` is the currently-active tool/weapon.
- **`scripts/ui/Minimap.gd`** — 180×180 pixel hex overview. `_draw()` renders discovered hexes (filled), current player position (cross), rifts (yellow dot with cross), Riftspire (orange star), towns (faction-colored circle). Faction colors are stable per-name via a codepoint-sum hash. Reads GameState + RiftRunner.
- **`scripts/ui/InventoryScreen.gd`** — 6×5 grid (30 slots). Right-click on a slot opens a PopupMenu (Use / Add to Hotbar / Drop). "Add to Hotbar" writes the item_id into the first empty HUD hotbar slot. "Use" only handles `bandage` in Phase 2; other items just print a placeholder message.

### Modified
- **`scripts/HubWorld.gd`** — `_setup_hud` instantiates the new HUD and hides the old `CharInfoBar` (the HUD has its own top bar). `_tick_hover_tooltip` still runs; added `_hud_minimap_tick` (refresh minimap every 1s). `_resolve_hotbar_tool` looks up the hotbar's selected item in `data/tools.json` and returns the tool data (used by `_try_start_gather`).
- **`scripts/GameState.gd`** — fixed pre-existing references to `SaveManager` as a class. Replaced `var sm: SaveManager = SaveManager` with `var sm: Node = get_node_or_null("/root/SaveManager")` (SaveManager has no `class_name`; using it as a type annotation was a latent parse error that became visible when Phase 2 added new autoload-dependent code).
- **`scripts/CharacterSelection.gd`** — same SaveManager type fix on line 487.
- **`project.godot`** — added `ProgressionManager="*res://scripts/ProgressionManager.gd"` to the autoload list.
- **`validate_scripts.gd`** — added 7 new scripts to the script list.

## 3. Key Decisions & Rationale

- **Two new autoloads (ProgressionManager, LootRoller) added in one phase.** They're tightly coupled (LootRoller writes to ProgressionManager) and both are pure data — no UI. Splitting them across phases would just add migration overhead.
- **Hotbar data stored as `Array[String]` of length 10, empty slots = "".** Simple, no embedded metadata. Phase 4's EquipmentManager will replace the "selected slot = equipped" model with a proper MainHand swap, but the hotbar array stays the same shape.
- **Minimap drawn via `_draw()` on a Control, not a SubViewport.** The SubViewport approach is more accurate (renders actual world content) but is overkill for 180×180 pixels. Custom _draw is faster and gives total control over the visual style.
- **Faction color = stable hash of faction name → hue.** A simple deterministic color per faction (no need for a factions.json color field). If we add 50 factions, they'll all get distinct-enough colors.
- **InventoryScreen "Use" only handles bandage.** Other consumables will be added in later phases as their effects are implemented. The infrastructure (right-click context menu, Add to Hotbar, Drop) is in place.
- **Pre-existing SaveManager bug fix included.** It was a latent issue: `var sm: SaveManager = SaveManager` worked in the loose GDScript parser when nothing depended on the type, but broke when Phase 2 added new autoload-dependent code. Fix: use `Node` type and `get_node_or_null`. Cleaner than adding `class_name SaveManager` (which would conflict with the autoload singleton of the same name).
- **HP/MP bars always show 100/100 in Phase 2.** Real HP/MP system lands in Phase 4 with EquipmentManager. The bars exist so the HUD layout is in place; they'll start working when the player has stats.
- **Old `CharInfoBar` hidden, not removed.** The HUD has its own top bar with the same info. Keeping the old one as a hidden node means I don't have to touch the existing `HubWorld.tscn` scene file.

## 7. Context That Must Survive

- **Tile cell size is 24×24.** Maps are 512×512 cells.
- **Atlas is 4 rows** (ground=0, debris=1, vegetation=2, blocked=3).
- **JSON load via `data` property:** `if "data" in raw: data = raw.data` (works for JSON instances and pre-cast Dictionaries).
- **Backups folder is excluded** from Godot (`backups/.gdignore`) and from git (`.gitignore`).
- **GDScript strict-type narrowing:** after `if bucket is Dictionary:`, `elif bucket is Array:` is a compile error. Use a fresh `var bucket = data.get(...)` if you need both checks.
- **SaveManager has NO `class_name`.** Use `var sm: Node = get_node_or_null("/root/SaveManager")` and call methods on the instance. Never use `SaveManager.foo()` as a static call, never use `var x: SaveManager`. The autoload is a Node singleton; the script's class identity comes from the file name + extends declaration, not class_name.
- **Hotbar slot data:** `Array[String]` of length 10; "" = empty. `select_slot(i)` highlights and emits `slot_selected`. `equipped_item_id` is the public read.
- **Hotbar keys:** KEY_1..KEY_9 map to slots 0..8; KEY_0 maps to slot 9. Implemented in `_unhandled_key_input`; consumes the event with `get_viewport().set_input_as_handled()`.
- **Minimap** is a `Control` with custom `_draw()` — no SubViewport. Reads GameState's `discovered_hexes`, `get_player_position`, world_data (riftspire_hex_key), RiftRunner.get_rifts_for_world. Stable faction colors via codepoint-sum hash → HSV hue.
- **HP/MP bars are placeholders** in Phase 2. Always 100/100. Real damage comes in Phase 4.
- **LootRoller.roll_and_apply** requires both InventoryManager and ProgressionManager to be available. Pass null to skip either.
- **Old `CharInfoBar` is hidden, not removed.** It's still in the scene tree. Phase 8 (UI rewrite) can remove it cleanly.
- **InventoryScreen right-click** uses `_marker_nodes`-style dict keys for context. Only `Use` for `bandage` is wired; other items print a placeholder.
