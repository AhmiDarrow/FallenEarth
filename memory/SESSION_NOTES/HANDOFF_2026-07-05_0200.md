## 3. Key Decisions & Rationale

- **No tool gating in Phase 1:** Without an EquipmentManager, there's no UI to equip a tool. `HarvestNode.try_gather` supports proper tool checking (stone_pickaxe only mines iron_outcrop, not copper) and the smoke test exercises it, but `HubWorld` calls it with a wildcard `"*"` harvests list so bare-hands works. Phase 4 introduces EquipmentManager + hotbar; the wildcard fallback can then be removed.
- **No respawn for floor pickups:** Sticks and stones are single-shot per map. The 262k-cell Ash Wastes map gets ~5k-7k sticks and ~7k-10k stones — plenty to find. If F5 shows the world feeling empty, we'd add respawn or increase density in a follow-up.
- **High node density:** total density across all categories is ~10% of walkable cells. Ash Wastes gets ~16k resource nodes. This may feel crowded; Phase 1 is "make the system work" first. F5 will tell. If it looks busy we'd trim densities.
- **JSON load via `data` property:** Godot 4.3 `load("res://file.json")` returns a JSON instance, not a Dictionary. We have to access `.data`. Helper `_load_json_dict` in `smoke_resource_nodes.gd` and inline pattern in `LocalMapGenerator` and `InventoryManager` use `if "data" in raw: data = raw.data` (works for both JSON instances and pre-cast Dictionaries).
- **Cached JSON loaders in LocalMapGenerator:** `LocalMapGenerator.generate` runs every hex entry, so `_load_biome_resource_nodes` and `_load_floor_pickup_densities` are cached by file mtime. Avoids re-parsing 262k cells of node data per hex.
- **FloorPickup uses queue_free on collect:** Single-shot per map. The pickup's sprite is gone immediately on walk; the InventoryManager has the item. Memory-clean.

## 7. Context That Must Survive

- **Tile cell size is 24×24.** Maps are 512×512 cells.
- **Atlas is 4 rows** (ground=0, debris=1, vegetation=2, blocked=3). BLOCKED has full-cell collision on physics layer 0.
- **`@onready` doesn't fire in `SceneTree._initialize`** for a freshly added node — smoke tests use `await process_frame` after `add_child` to ensure `_ready` has run. Real game scenes don't have this issue.
- **Backups folder is excluded** from Godot (`backups/.gdignore`) and from git (`.gitignore`).
- **JSON load via `data` property:** in Godot 4.3, `load("res://file.json")` returns a JSON instance, not a Dictionary. Access via `.data`. Pattern: `if "data" in raw: data = raw.data`.
- **InventoryManager inventory is non-persistent for Phase 1.** GameState.SaveManager will be extended in Phase 8 to read/write the inventory dict. For now, inventory resets on game close.
- **HarvestNode bare-hands wildcard `"*"`.** HubWorld passes a tool with `harvests: ["*"]` when no tool is equipped. Phase 4 (EquipmentManager) makes tool equipping real; remove the wildcard then.
- **Floor pickups don't respawn.** Single-shot per map. Sticks and stones are plentiful (5k-10k per map).
- **Node density totals ~10% of walkable cells.** Ash Wastes: ~16k nodes + ~8k pickups. May feel busy in F5.
- **Gather timer is gated by `Input.is_key_pressed`** + the existing input system in HubWorld. The `E` key was added to `_unhandled_input`; matches the existing WASD/M/ESC pattern.
- **InventoryManager signals are NOT yet connected to anything.** Phase 2 will wire them to HUD; Phase 8 will wire to save/load.
