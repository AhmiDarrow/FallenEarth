## Bug

User: "I cannot see the mobs nor do they initiate combat if I am running into them"

## Two Root Causes Found

### 1. Scene parse error (the original fix)

`scenes/HubWorld.tscn` (and 3 test scenes) had an `[ext_resource]` for
`res://scripts/procedural/Entity3DViewport.gd` (id="2_ev") that referenced a
file deleted from the working tree. The 18 procedural files
(Entity3DViewport, EntityAnimator, EntityLODManager, EntityPool, EntityPreviewer,
EntityVisualComponent, FaceCamera3D, MaterialLibrary, Palette, ParticleEmitters,
PrimitiveMeshLibrary, ProceduralCharacter, ProceduralEntityGenerator,
ProceduralMob, ProceduralRenderer, ProceduralTile, RiftPalette, RiftVisual)
were intentionally removed (migrating from procedural 3D entities to flat PNG
sprites) but the scene references were not cleaned up.

Result: `Parse Error: [ext_resource] referenced non-existent resource` logged
on every F5 launch. Even though the root `HubWorld.gd` script still attached
and `_ready()` could still run, downstream rendering paths didn't complete
reliably. Mobs were not seeded/visible and combat triggers were not firing.

### 2. Mob PNGs were flat 1-color blobs (the second fix)

After fixing the parse error, mobs were technically rendering — but every
`assets/mobs/*.png` was a single solid color with zero internal shape detail.
Analysis:

```
charnel_stalker.png: 1 unique colors {(128, 128, 128)}
ashveil_grazer.png:  1 unique colors {(196, 168, 106)}
glimmer_swarm.png:   1 unique colors {(0, 255, 159)}
... (all 27 mobs were 1-color)
```

`generate_mob_sprites.py` was drawing the body, head, legs, and outline ALL in
the same base color, so they merged into one solid silhouette. The eye/details
were also drawn in the same color so they disappeared. At 32×32 rendered size
(scale 0.5 in MobVisual) these flat blobs blended into the terrain.

## Fixes Applied

### Fix 1 (commit 69d53c2)

Removed the orphaned `[ext_resource]` and the `Entity3DLayer` / `Entity3DDisplay`
nodes from:

- `scenes/HubWorld.tscn` (critical)
- `scenes/EntityAnimatorTest.tscn`
- `scenes/Phase5ItemsPropsTest.tscn`
- `scenes/Phase6PerformanceTest.tscn`

### Fix 2 (uncommitted, this session)

Fixed `generate_mob_sprites.py`:

- Properly calls `_shade(base_color, 0.25)` to get a dark outline color before
  drawing the inflated shape, then draws the body at normal size on top. The
  previous version drew the inflated + normal shapes BOTH in the base color,
  so the outline never actually existed.
- Added per-archetype shape functions with internal contrast:
  - `_quadruped` / `_insectoid` / `_floater` / `_behemoth` / `_mechanical` /
    `_aberrant`
  - Each draws body, head, eye/mandibles/spikes, and (for non-floater) legs in
    body color, with a 2px outline in a 25%-brightness shade of the base color.
  - Floater and mechanical get an extra "glow" / "visor" highlight in a
    brightened shade (1.4x) for visual interest.
- `_draw_outline` helper defined but the actual main() does the inflation
  inline (cleaner).
- Also auto-generates sprites for any mob in `data/mobs.json` whose
  `sprite_id` is not in `data/mob_sprites.json` (catches the 5 rift_only mobs
  + a couple of overworld ones that were missing from the sprite registry).

Regenerated all 27 mob PNGs. After fix:

```
ashveil_grazer.png  → body+head+eye+4 legs visible, dark outline around silhouette
charnel_stalker.png → same (1 unique body color + dark outline)
lifecycle_horror.png → distorted mass + tendrils + 5-eye cluster + outline
```

All 27 PNGs now have a clearly visible body outline + body parts. ASCII
