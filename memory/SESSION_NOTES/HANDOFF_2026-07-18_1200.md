  - `scripts/mob/MobSpawner.gd` — added `MobDataRef` preload; replaced all references
  - `scripts/mob/OverworldMobManager.gd` — added `MobDataRef` preload; replaced all references
  - `scripts/mob/OverworldMobPool.gd` — added `MobInstanceRef` preload; replaced all references
- **Fixed smoke tests**: Updated `smoke_audio.gd` path from `CombatLevel.gd` → `CombatLevel3D.gd` (file was renamed); fixed type hints in `smoke_phase2.gd` and `smoke_tile_system.gd`

## 2. Key Decisions & Rationale

1. **Preload aliases over autoloads**: Rather than making WorldGenerator etc. autoloads (which would change runtime semantics), added `const` preloads with aliased names (`MobDataRef`, `HotbarRef`, etc.) and used those as type hints. This is the least invasive fix for the GDScriptCache stricter resolution in 4.4+.
2. **No `--convert-4x` run**: The tool exists but timed out and the project had pre-existing compilation issues. For minor 4.x version bumps, the main change needed is `config/features` in `project.godot`, which was done manually. Scene/resource files are forward-compatible.
3. **Kept both Godot executables**: `Godot_v4.3-stable_win64.exe` remains in the project dir (gitignored) for side-by-side comparison if needed.
