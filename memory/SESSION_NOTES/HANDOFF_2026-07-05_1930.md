## Key Design Decisions

1. **Room size 12×10 at 24px/cell** — small enough for a single room, large enough for NPCs + exits
2. **Color-coded tiles** — no sprites needed for Phase B; walls dark, floors lighter, exits green
3. **NPCs as colored dots** — simple but functional; future phases can replace with character sprites
4. **Grid-based collision** — no physics, just `is_wall()` check on the string grid
5. **Room transitions instant** — no animation; old RoomView freed, new one created
6. **Backward compatible** — old `Settlement.gd` still exists (not deleted), but SettlementManager no longer loads it

## Context That Must Survive

- RoomView.gd uses `class_name RoomView extends Node2D` — loaded via `set_script(load(...))` on a fresh Node2D
- SettlementInterior creates rooms inside a `RoomContainer` Node2D child, not directly as its own children
- `data/settlement_rooms.json` grid format: "#" = wall, "." = floor, "X" = exit. Must be exactly 12 chars per row, 10 rows
- Exit targets starting with "_" (e.g. "_settlement_exit") are treated as special (not looked up in rooms dict)
- NPC interaction by role: only "trader" and "quest_giver" open real UIs; all others show greeting dialog
- The old `Settlement.gd` and `Settlement.tscn` still exist in the project but are no longer loaded by SettlementManager
