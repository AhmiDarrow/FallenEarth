# Session 2026-07-05_2217 — Audio wiring hotfix

## Summary

Fixed "sounds does not work" — two root causes + one polish.

1. **All 12 audio `.import` files had `loop=false`** → flipped to
   `loop=true` (ambient loops + music themes both need to repeat).
   Wiped `.godot/imported/*.oggvorbisstr` cache and re-ran
   `godot --headless --import` so the binary cache reflects the
   new flag.

2. **MusicManager + AmbientAudio autoloads were never called.**
   The systems existed but no scene's `_ready` invoked them.
   Wired 6 scenes:
   - `MainMenu` → `main_menu` music, no ambient
   - `HubWorld` → `exploration` music + biome ambient (with
     `map_biome()` helper to translate the 10 world biomes into
     the 6 ambient bed keys). Refreshed on every hex cross in
     `_try_cross_edge`.
   - `SettlementInterior` → `settlement` music + ambient
   - `RiftInstance` → `rift` music + drone
   - `TacticalCombat` → `combat` music, mutes ambient
   - `WorldMapScreen` → `exploration` music, mutes ambient

3. **`OptionsMenu` sliders only persisted to disk** — added
   live `set_volume` calls so changes are audible immediately.
