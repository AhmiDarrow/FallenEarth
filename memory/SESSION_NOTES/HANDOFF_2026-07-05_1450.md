## What Was Accomplished

### A1: Extended `data/towns.json` with building_types
- Added `building_types` dictionary with 9 building definitions: tavern, trader, worktable, armor_table, blacksmith, quest_board, faction_hq, auction_house, arena
- Each building has: `w`, `h` (cell dimensions), `role` (interaction type), `sprite` (PNG ID), `label` (display name)
- Templates unchanged (small_outpost=3 buildings, medium_settlement=6, large_hub=9)

### A2: Generated 9 building sprites
- Created `tools/generate_building_sprites.py` — procedural PIL generator
- Output: `assets/sprites/buildings/{id}.png` — top-down view at 24px/cell
- Sizes: quest_board=24x24, worktable/armor_table/blacksmith=48x48, tavern/trader/auction_house=72x72, faction_hq=96x72, arena=120x120

### A3: Town layout generator in `LocalMapGenerator.gd`
- `_get_town_for_hex(q, r)` — looks up town data from GameState (with static cache)
- `_generate_town_layout(rng, town_data, terrain, occupied, spawn)` — algorithm:
  1. Clears circular clearing (radius 15) at center
  2. Places buildings evenly spaced on ring road (radius ~19)
  3. Marks footprints as TERRAIN_BLOCKED + occupied
  4. Draws debris paths from building entrances to clearing
- `_compute_town_boundary(structures, center)` — returns Rect2i bounding box
- `generate()` now populates `map_data["settlement"]` with structures, town_data, and boundary

### A4: SettlementBuilding scene + rendering
- New `scripts/SettlementBuilding.gd` + `scenes/SettlementBuilding.tscn`
- Node2D with Sprite2D + Label, loads PNG from `assets/sprites/buildings/`
- Methods: `setup(data)`, `is_cell_inside(cell)`, `get_entrance_cell()`, `get_role()`, `get_building_id()`
- `LocalMapView._populate_buildings()` renders buildings on settlement_layer
- `LocalMapView.get_building_at(cell)` — hit-test for interaction
- `LocalMapView.get_map_data()` — returns current map_data for HubWorld access

### A5: Fixed station_layer bug
- Added `station_layer = get_node_or_null("StationLayer") as Node2D` to `LocalMapView._ready()`
- Cooking tables now actually appear on the map (was dead code before)

### A6: Town boundary + mob exclusion
- `map_data["settlement"]["boundary"]` stores Rect2i of town area
- `HubWorld._seed_local_mobs()` now checks boundary — mobs skip cells inside town

### A7: Building interaction in HubWorld E-key
- `_adjacent_building()` — checks player's cell + 8 neighbors for SettlementBuilding
- `_interact_building(building)` — dispatches by role:
  - `trader` → enters settlement (opens shop via SettlementManager)
  - `quest_giver` → opens MissionBoardInterface
  - Other → enters settlement interior
- E-key priority: settlement interior > base > cooking table > building > gather

### A8: Smoke test
- `tools/smoke_settlement.gd` — 10 test groups, all green
- Tests: towns.json loading, 3 template sizes, terrain marking, occupied marking, boundary, scene load, LocalMapView integration, sprite existence

## Key Decisions

1. **Buildings are TERRAIN_BLOCKED** — player walks around them, interacts at entrance cell
2. **Clearing radius 15** — overlaps spawn pocket (radius 24) so town is centered
3. **Ring road placement** — buildings evenly spaced around clearing, paths connect to center
4. **Existing SettlementNode preserved** — stays as town center marker (★); buildings are additional sprites
5. **Backward compatible** — empty `structures[]` = old behavior for non-town hexes
